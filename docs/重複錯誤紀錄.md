## ★高頻・最常重犯

### E001 — 回 Kevin 訊息忘了走 Discord reply 工具（純文字打終端）
- **症狀**：把要給 Kevin 看的內容直接輸出成終端文字，沒呼叫 `mcp__plugin_discord_discord__reply` → Kevin 在 Discord 完全收不到，他問「都沒收到」。

### E002 — FB「發佈」鈕狂點 → 發出重複貼文
- **症狀**：gstack 點 FB 發佈鈕，click registered 但 dialog 不關、貼文沒立刻出現 → 連點多次 → 結果發了重複貼文（2026-06-24 套房文疑似 1 完整＋1 只有圖）。
- **根治**：①發佈鈕只點 1 次，等久一點（5-7s）再核對；②發完一定去 `facebook.com/me` 數關鍵字出現次數確認發了幾則；③有重複要請 Kevin 授權才刪（刪文不可逆）。

## 其他踩雷

### E003 — gstack upload 圖片路徑限制
- inbox 路徑（`C:\Users\USER\.claude\channels\...`）被擋「Path must be within」。**圖要先 cp 到 `C:\Users\USER\claude-bot` 或 `AppData\Local\Temp` 再 upload。**

### E004 — gstack 中途 re-snapshot 會把已上傳的圖洗掉
- 上傳圖後若又動 DOM/snapshot，已塞的圖會掉，要重傳。**上傳圖→確認→盡快發佈，中間別亂動。**

### E005 — claude-in-chrome 本機被權限擋
- `screenshot`/`get_page_text` 回 `Permission denied`（瀏覽器寫入權限要 agent 重啟才生效）＋本來就塞不了圖。**發文改走 gstack `browse connect`（接 real Chrome、已登入）。**

### E006 — Chrome cookie 在 Windows 匯不出
- `cookie-import-browser chrome` 回 `DPAPI decryption failed`（Chrome app-bound 加密）。**別靠匯 cookie，改用 `browse connect` 接已登入的 real Chrome。**

### E007 — 美術/視覺是弱項
- Kevin 評「做的都很醜」。**工程佔位版可接受，但正式上線前視覺交給設計師（Lumi）或認真走設計流程，別拿工程佔位美術當成品交付。**
