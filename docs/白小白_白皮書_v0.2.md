# 白小白・你的貼文小助手 — 產品白皮書 v0.2

> 產品名「**白小白**」（暱稱：小白／白白），副標「你的貼文小助手」。一個會跟你聊天、幫你想內容、提醒你發的擬人化小助手。
> **品牌歸屬：好厝安居 出品的「貼文輔助系統」**（套用好厝 VI、自帶信任感）。
> 本文件為內部白皮書草案，作者：阿凱（為 Kevin 撰）。日期：2026-06-24。定價、規則皆可調。

---

## 0. 一句話定位

**「你不用會行銷、不用想發什麼，拍張照、跟白小白聊兩句，它用你的口氣每天幫你寫好一篇、提醒你發。」**

賣的不是「文案生成」，是「幫你把社群持續經營這件事做完」的結果——而且是用一個**有溫度的小角色**陪你做。

---

## 1. 為什麼做這個（問題）

剛起步、想經營社群的小白商家（美甲、小吃、手作、二房東、個人接案…），每天面對三個過不去的坎：

1. **不知道發什麼** —— 對著空白頁發呆。
2. **不知道為什麼發** —— 亂發、沒策略、看不到成效。
3. **撐不過兩週就斷更** —— 三分鐘熱度，帳號荒廢。

市面上的通用 AI（ChatGPT／豆包／千問）能「生一段字」，但**結構上不會**：連到你的社群幫你發、記住你的帳號經營到哪、替你天天持續。工具再強，最後一哩還是要你自己扛——而小白扛不動。

**白小白要做的，就是這條「大公司懶得做、小白又最需要」的最後一哩。**

---

## 2. 目標客群

- **主要**：台灣、剛開始經營社群的個人/微型商家（不會行銷、沒預算請小編、用手機、發個人 FB 為主）。
- 典型畫像：美甲師、小吃/飲料店、手作賣家、寵物美容、個人工作室、二房東、保險/房仲個人戶。
- **共同點**：有產品/服務、有手機、會拍照，但「不會寫、不會排、撐不久」。

---

## 3. 價值主張與護城河（三根柱子，缺一即垮）

「生內容」本身沒有護城河（人人叫 AI 都會）。白小白的差異化在通用 AI **結構上做不到**的三件事：

1. **執行閉環** —— 直接接到你的社群：排好、到點提醒、一鍵/半自動發。把「每天自己複製貼上」變成「點一下」。
2. **持續運轉＋會學習** —— 是一直在跑的系統：記錄每篇成效、第 15 天自動接著排、好的提頻、差的換掉，越用越準。通用 AI 是一次性問答、用完就忘。
3. **零門檻** —— 把產業 know-how、語氣、爆文公式**內建**好。使用者不用會下指令，**拍張照就有成品**。

比喻：
- 通用 AI＝什麼食材都有的大超市（你還是得自己會煮、天天煮）。
- 白小白＝每天把煮好的便當送到你家、還提醒你吃（針對「顧社群」這一件事幫你做完）。

## 4. 產品架構（三層）

```
📱 殼 App（WebView 殼，上架商店）
    └─ 原生功能：推播通知、相機拍照、分享到 FB/IG
🌐 網頁 App（產品本體・我們的品牌與介面）
    └─ 產業卡片 / 語氣蒸餾 / 14天排程 / 看圖生文 / 發文閉環 / 戰績紀錄
🧠 後台模型引擎（可替換的燃料，客人看不到）
    └─ 呼叫某顆模型 API（Claude / 千問 / 豆包…擇優；我們的提示詞與規則包在外面）
```

設計原則：
- **模型是可替換的燃料**，那層殼（產業 know-how、語氣蒸餾、發文閉環、品牌）才是護城河。
- **只寫一套網頁當大腦**，殼只是把它包成 App → 開發省、維護省、跨 iOS/Android。
- 客人永遠看不到「Claude」「千問」等字眼，用的是「白小白」這個品牌。

### 為什麼不是 LINE、不是純網頁、不是叫客人裝 AI
- ❌ **LINE OA**：每月免費推播額度少（約 200 則）、超量加購極貴；每天推一篇 = 成本自殺；API 串接麻煩、寄人籬下。
- ❌ **純手機網頁**：iPhone 的 Web Push 要先「加到主畫面」才有推播、較弱，且少了「正派 App」的信任感。
- ❌ **叫客人在自己電腦裝一套 AI/Claude**：門檻太高、不可能。
- ✅ **WebView 殼 App**：客人下載「一個正常的 App」（有 icon、在商店、看起來正派），**原生推播穩**、能用相機與分享；我們只維護一套網頁。
  - 實務提醒：Apple 審核會嫌「純空殼功能太少」→ 殼內放原生推播＋相機＋分享即可過審（標準操作）。

### 4.1 技術選型（起手式，都有免費額度）
- **前台（畫面）**：Netlify 或 Vercel — 免費、好部署。（「不好修改」只是因為要改原始碼再重新部署，是正常開發流程。）
- **後台（大腦／API／推播）**：Netlify/Vercel serverless 函式，或 Render／Railway。
- **登入＋資料庫＋檔案儲存**：**Supabase**（免費額度大、內建帳號登入＋Postgres＋storage，最省事）。
- **模型 API**：由後台呼叫。
- 🔐 **鐵則**：模型 API 金鑰**只能在後台呼叫，絕不放前台網頁**（會被偷、帳單爆掉）。所以不可能「只丟一個靜態站」了事。

## 4.5 擬人化角色「白小白」（一隻小白白貓 🐱）

白小白不只是工具，是一個**會跟你聊天的小角色**——介面即角色，降低小白「我不會用 AI」的恐懼。

- **角色設定（Kevin 2026-06-24 定）**：一隻**小白白貓**，個性親切、會鼓勵你、有點俏皮，像「懂行銷的好朋友」。MVP 先用靜態小白貓。
- **★第二階段：養成／互動系統（roadmap，留存引擎）**：
  - 連續發文／達標 → 小白貓**進化解鎖**（小貓 → 貓娘）。
  - **造型商城**：解鎖/收集衣服、道具、配件幫小白貓打扮。
  - **互動對話**：養成式陪伴，每天回來不只是發文，是「照顧你的小白貓」。
  - 戰略意義：把「每天發文」這件容易斷掉的事，變成「捨不得斷的養成遊戲」→ 直接補強護城河三柱中的「持續性」。
- **互動方式**：客人跟它「聊天」而不是「下指令」。例：丟張照片＋「幫我發這個」→ 白小白回「收到～這款好美！我幫你寫一篇引流的，你看看 👇」。
- **情感設計**：每天提醒像朋友提醒（「今天還沒發喔，我幫你準備好囉」），不是冷冰冰的通知。
- **品牌資產**：角色 = 記憶點 = 可做貼圖/吉祥物/週邊，行銷時白小白本身就是話題。
- **語氣界線**：白小白對「客人」說話的語氣（親切俏皮）≠ 白小白「幫客人產出的貼文」語氣（要照客人自己的語氣蒸餾），兩者分開，別混。

## 5. 核心功能模組

### 板塊一：登入與綁定（安全第一）
- 用我們自己的帳號系統登入（手機/email），**絕不要客人的 FB 帳號密碼**。
- 「綁定」採官方授權或本機分享流程，讓客人**零帳密外洩感**。
- 發文方式（依客人方案）：
  - **半自動（主力）**：產好「圖＋文」→ 客人一鍵複製/喚起 FB 分享 → 自己按發佈（最後一下＝安全閥，他帳號發什麼自己把關）。
  - **全自動（進階/第二階段）**：電腦瀏覽器外掛，排程到點自動發（含圖）。需電腦開著，給少數要全自動的進階客。

### 板塊二：選語氣／文案模板
- **A. 產業卡片（內建）**：勾選所在產業 → 套我們做好的語氣與題材卡片，開箱即用。
- **B. 蒸餾自己**：請客人丟自己的舊文 → 系統學他的語氣。
  - ⚠️ **責任聲明（必須顯示且勾選同意）**：建議蒸餾「您本人」的文案；若放入他人文案，相關著作權/責任由使用者自負，本服務不負責。

### 板塊三：自動產出內容
- **直接排 14 天內容**，每篇**標注用意**（見 §6 用意分類）。
- **看圖生文**：客人拍照丟進來 → 系統看懂圖 → 用他的語氣＋指定用意生對應文案（例：美甲新圖案 → 引流文案）。
- **自輸入想法**：客人也可打一句自己的想法 → 同樣在此板塊產出。
- 每篇可選「要不要配圖、配哪張」。

### 板塊四：持續與成長（系統的靈魂）
- 發文後回收成效（讚/留言/分享）→ 記錄戰績。
- 第 15 天自動接續排程；好的公式提頻、差的換題材。
- 定期「回顧」：抓最好/最差公式，調整下一輪日曆。

## 6. 文案細節規則（核心資產）

> 這是白小白真正值錢、別人抄不走的部分。模型可換，這套規則是我們的。

### 6.1 語氣規則（蒸餾 & 套用）
- 生成任何草稿前，**先載入該客戶的語氣設定**（產業卡片或蒸餾結果），不像他的語氣就重生成（**公式 < 語氣**）。
- 蒸餾項目：句長與節奏、慣用語/口頭禪、標點與 emoji 習慣、稱呼方式、收尾風格、不能講的話。

### 6.2 反 AI 腔鐵則（真實感）
- **嚴禁抽象空詞**：護城河、本質、賦能、底層邏輯、真正的 X、長期主義、降維、儀式感、未來已來…出現即重寫。
- 禁 staged 開場（「你有沒有想過…」式假開場）、禁 over-narrate（過度自我解說）。
- 真實感來自：真產品、真細節、真動作、口語、自嘲——不是形容詞堆疊。
- 標點隨「該客戶語氣」走（沉穩型少驚嘆/emoji；活潑型可適度）——不是一刀切。

### 6.3 用意分類（每篇必標其一）
| 用意 | 目的 | 典型公式 |
|---|---|---|
| **介紹** | 讓人知道你是誰/做什麼 | 自我故事、服務介紹、品牌理念 |
| **Hook/吸睛** | 衝停留、博眼球 | 反差數字、反命題、痛點開場 |
| **知識/權威** | 建立專業信任 | 行業內幕、迷思破解、教學乾貨 |
| **故事/信任** | 拉近、建立情感 | 真實案例、創業脆弱、客戶故事 |
| **引流** | 要私訊/留言/名單 | 痛點→好處→留言關鍵字 CTA |
| **促銷/催單** | 直接成交 | 限量/限時/名額、優惠 |
| **互動** | 衝留言、養觸及 | 開放式提問、二選一、投票 |

- 一輪 14 天**配比平衡**（不要全是促銷）；同一用意短期內不重複題材。

### 6.4 公式庫（依用意取用，可跨產業套）
- **痛點 Hook 型**：段1 戳痛點 → 段2 點出真正原因 → 段3 帶出方案 → 收一個問句。
- **知識權威型**：反直覺破題（「越 X 反而越 Y」）→ 拆原理 → 對應到產品 → 安全/負責提醒。
- **故事信任型**：真實場景 → 一個轉折/反差 → 收在讓讀者自問的問題。
- **引流型**：共鳴 hook → 三個好處 → 「留言『關鍵字』我私你」CTA（用留言關鍵字，不在正文導外連）。
- **促銷型**：稀缺（名額/時間）+ 行動指令，簡短有力。
- **互動型**：一句開放式提問或二選一，鼓勵留言。

### 6.5 看圖生文規則
- 先辨識圖中：主體（產品/服務/場景）、風格、亮點、情境。
- 對齊「客戶語氣 + 指定用意」生文，不脫離圖的真實內容（不杜撰圖中沒有的東西）。
- 涉及效果/療效/專業宣稱，加**安全/免責提醒**（如兒童用品提醒大人陪同、美容類不誇大療效）。

### 6.6 平台規則
- **FB / Threads（脆）正文絕不放外部連結**（會被降觸及）；要導流用「留言關鍵字 → 私訊/精選留言放連結」。
- **IG 必配圖**；純文字題材改發脆/FB。
- 每平台**重新生成**，不同一段複製貼上（語氣/長度/節奏各自調）。
- 一天節奏建議：圖文題材分配到 IG（午）/FB（晚）/脆（夜），沒料的時段寧可少發不硬擠。

### 6.7 安全與品牌邊界
- 不碰政治/社會爭議、不點名同業、不灑狗血、不做誇大不實宣稱。
- 引流爆量時**不手動大量私訊**（FB 易判異常封號）→ 改公開精選留言放連結。

## 7. 商業模式（Kevin 2026-06-24 定）

- **免費版**：一天**只能自動 1 次**（每日 1 篇供稿＋提醒）。當作引流與試用，讓人先體驗到價值。
- **付費階級**：功能/額度分級（如一天多篇、全自動發、IG、多帳號…）→ **後續再討論細部定價**。
- 成本結構健康：只付「伺服器＋模型 API」，**沒有 LINE 那種按則收費的無底洞**。
- 早期變現驗證：先用免費版養使用者＋驗證留存，再開付費階級。

## 8. 風險與合規

- **平台條款**：個人塗鴉牆自動代發踩 FB 條款 → 主力走「半自動（使用者自己按發佈）」規避；全自動僅限本機外掛、使用者自有環境。
- **帳密安全**：絕不蒐集/儲存客人 FB 帳密；綁定走官方授權或本機分享。
- **內容責任**：蒸餾他人文案的著作權/責任由使用者自負（需勾選同意條款）。
- **模型依賴**：模型可替換，避免單一供應商鎖死（成本/政策風險分散）。

## 9. MVP 範圍（第一階段，先驗證）

**目標：用最小成本驗證「小白願不願意付錢」。**

- 平台：先做 **FB 個人頁 + 脆（Threads）**，半自動。
- 產業卡片：先做 **3 個（Kevin 定）：美甲、包租代管、房仲**（後兩個是 Kevin 本行，題材與語氣最到位；美甲驗證跨產業通用性）。
- 功能：登入 → 選產業/蒸餾語氣 → 產 14 天（標用意）→ 看圖生文 → 一鍵複製/分享 → 推播提醒 → 記錄戰績。
- 外殼：先 PWA 或簡易殼 App 測試使用流程，過審細節隨後處理。
- **暫不做**：全自動外掛、IG、團隊版、金流自動化（可先手動收款驗證）。

**第二階段**：電腦外掛全自動、IG、更多產業卡片、金流自動化、上架商店。
